# 网络 知识点

## 1. HTTP 协议

- HTTP/1.0：每次请求都需要建立连接
- HTTP/1.1：支持持久连接，流水线传输
- HTTP/2：多路复用，服务器推送，头部压缩
- HTTP/3：基于 QUIC 协议，更快的连接建立

## 2. HTTP 方法

- GET：获取资源
- POST：提交数据
- PUT：更新资源
- DELETE：删除资源
- PATCH：部分更新资源
- HEAD：获取头部信息
- OPTIONS：获取服务器支持的方法

## 3. HTTP 状态码

- 1xx：信息性状态码
- 2xx：成功状态码（200 OK, 201 Created）
- 3xx：重定向状态码（301 永久重定向, 302 临时重定向, 304 未修改）
- 4xx：客户端错误（400 错误请求, 401 未授权, 403 禁止访问, 404 未找到）
- 5xx：服务器错误（500 内部服务器错误, 503 服务不可用）

## 4. HTTPS

- 定义：HTTP + SSL/TLS 加密
- 作用：保证数据传输的安全性
- 证书：由 CA 颁发，验证服务器身份
- 握手过程：客户端和服务器协商加密算法和密钥

## 5. 跨域问题

- 同源策略：协议、域名、端口都相同
- 跨域解决方案：
  - CORS（跨域资源共享）
  - JSONP
  - 代理服务器
  - WebSocket
  - postMessage

## 6. 网络请求优化

- 减少请求数量：合并资源，使用精灵图
- 减少请求大小：压缩资源，使用 gzip
- 使用 CDN
- 缓存策略
- 预加载：`preload`, `prefetch`
- 延迟加载：图片懒加载

## 7. TCP/IP 协议

- TCP：面向连接的可靠传输协议
- IP：网络层协议，负责路由和寻址
- TCP 三次握手：建立连接
- TCP 四次挥手：关闭连接

## 8. 网络攻击

- DDoS 攻击：分布式拒绝服务攻击
- SQL 注入：注入恶意 SQL 语句
- XSS：跨站脚本攻击
- CSRF：跨站请求伪造
- 防范措施：输入验证、输出编码、使用 HTTPS 等
